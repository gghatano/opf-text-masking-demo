# OpenAI Privacy Filter 日本語適用性評価 実証実験計画書

## 1. 背景

個人情報を含む自由記述データの利活用において、個人識別子の検出・マスキング作業は大きな負担となっている。

特に以下のようなデータでは、構造化項目だけでなく自由記述欄に個人情報が含まれる。

* 医療データ（診療録、看護記録、退院サマリー）
* 自治体相談記録
* コールセンター応対履歴
* アンケート自由記述
* 研究用データセット

近年、OpenAIよりオープンウェイトのPII検出モデル「OpenAI Privacy Filter」が公開された。本モデルは個人情報の検出およびマスキングを目的としており、軽量かつオンプレミス実行可能である。

一方で、日本語データに対する性能や、医療・自治体分野における実用性は十分に確認されていない。

本実証では、日本語自由記述データに対する適用可能性を評価し、匿名加工・仮名加工業務の省力化への活用可能性を検討する。

---

# 2. 目的

以下を明らかにする。

1. 日本語自由記述データに対して実用的なPII検出性能を有するか
2. 医療・自治体領域特有の識別子を検出できるか
3. 追加学習による性能改善が期待できるか
4. 人手による匿名化作業の削減効果が見込めるか

# 5. アノテーション方針

## ラベル体系（2026-06-08 改訂・10ラベル）

> 旧18ラベルを整理。業務識別子4種は実質すべて個人IDのため `ID` に統合。担当者名・医師名は人名なので `PERSON` に統合。会社/学校/施設(病院含む)は所属組織として `ORGANIZATION` に統合。

### 直接識別子（基本PII）

| ラベル     | 例                          | 統合元 |
| ------- | --------------------------- | --- |
| PERSON  | 山田太郎 / 佐藤医師 / 担当者名     | 旧 STAFF を統合 |
| ADDRESS | 東京都千代田区                | |
| PHONE   | 090-XXXX-XXXX               | |
| EMAIL   | xxx@example.com             | |
| DATE    | 2025/01/01 / 2025年1月3日      | |
| ID      | 患者番号 / 被保険者番号 / 受付番号 / 会員番号 | 旧 業務識別子4種を統合 |

### 準識別子

| ラベル          | 例                | 統合元 |
| ------------ | ----------------- | --- |
| AGE          | 98歳              | |
| REGION       | ○○市              | |
| OCCUPATION   | 医師              | |
| ORGANIZATION | △△株式会社 / ○○高校 / ○○病院 | 旧 COMPANY/SCHOOL/FACILITY を統合 |

---

## アノテーション規則（2026-06-09 確定 #29）

正解ラベル(gold)の付け方を以下に統一する。検出評価の分母を安定させるため。

1. **スパン境界**: 敬称(様/さん/氏/医師 等)は**含めない**。本人名のみをスパンとする。
2. **REGION と ADDRESS の切り分け**: 完全住所(都道府県＋市区町村＋丁番地)=**ADDRESS**、地名単独(「○○市」等)=**REGION**。
3. **第三者・続柄**: 本人以外(長男/内縁の夫/担当ケアマネ等)が指す人物・属性も**対象**とする。ただし続柄語そのもの(「長男」等の関係を表す語)は対象外。
4. **多重該当時の優先**: 1スパン1ラベル。病院名は地域×施設で準識別子性をもつが **ORGANIZATION** とする。
5. **一致基準**: スパン一致は **char IoU≥0.5** で統一（#19 と整合）。

# 6. 評価指標

## エンティティ単位

* Precision
* Recall

> F1等の総合指標は**用いない**。見逃し(Recall)と過剰検出(Precision)はコストが非対称のため個別に評価する（2026-06-08改訂 #24）。

---

## ラベル単位

* PERSON
* ADDRESS
* ID
* ORGANIZATION
* AGE

ごとに算出（主要ラベル。10ラベル全体でも算出）

---

## 実務観点

### 漏れ率

検出されなかったPII件数

### 誤検出率

PIIでないものをPIIと判定した件数

### 作業削減率

人手確認工数削減割合

# 9. 成功判定基準

## 技術面

| 項目            |    目標 |
| ------------- | ----: |
| PII Recall（直接識別子中心） | 90%以上 |
| PII Precision | 85%以上 |
| 主要ラベル Recall / Precision | 90% / 85%以上 |

> Recall の主対象は直接識別子(PERSON/ADDRESS/PHONE/EMAIL/DATE/ID)。準識別子(AGE/REGION/OCCUPATION/ORGANIZATION)は参考測定。**F1は用いない**（2026-06-08改訂 #24）。

---

## 業務面

| 項目    |    目標 |
| ----- | ----: |
| 作業削減率 | 50%以上 |
| 見逃し率  |  5%以下 |

# 10. 想定リスク

| リスク        | 対応          |

| 誤検出増加      | 閾値調整        |
| 過度な期待      | 人手レビュー前提で運用 |

# 12. 今後の発展

本実証で有効性が確認できた場合、以下への展開を検討する。

1. 自治体向け個人情報ファイル簿作成支援
2. 医療情報匿名加工支援
3. 仮名加工情報作成支援
4. 自由記述マスキング支援サービス
5. AIを活用した匿名加工レビュー支援

特にユーザーの研究テーマとの関係では、「匿名加工作業の省力化」「自由記述データへの対応強化」の具体的な実装テーマとして位置付けられる。単なるNER評価ではなく、**匿名加工業務全体の工数削減効果を定量評価する実証**として設計するのが望ましいです。
